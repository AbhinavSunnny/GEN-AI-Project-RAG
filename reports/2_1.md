# Root Cause Analysis

The Fusion pipeline is more complex than Dense and Hybrid pipelines.

Fusion performs three retrieval operations:

1. Dense Retrieval

FAISS

2. Sparse Retrieval

BM25

3. Web Retrieval

DuckDuckGo Search

Example:

web_docs = web_search(
query,
max_results=5
)

The web retrieval stage introduces an external dependency.

Possible issues include:

* Rate limiting
* Temporary search failures
* Empty search results
* Network interruptions
* Query-specific failures

Any of these can cause a particular row to fail before:

results.append(...)

is executed.

When that happens:

the row is skipped

and no evaluation record is created.

---

# Why the Notebook Still Completed

The notebook itself did not fail.

Instead:

some iterations likely failed individually

while the outer loop continued.

As a result:

Total Expected Rows = 500

Missing Rows = 7

Generated Rows = 493

The notebook then reached:

"Fusion GPT4o Evaluation Completed"

and terminated normally.
